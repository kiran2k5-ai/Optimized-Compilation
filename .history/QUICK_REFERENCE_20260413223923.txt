PROBLEM 1: STUDENT CHANGES CODE AFTER SUBMISSION (CHEATING)
═══════════════════════════════════════════════════════════════════

SOLUTION: CodeAudit.php
├─ Creates SHA256 hash of every submission
├─ Stores timestamp, IP, user agent
├─ Detects if code was modified after grading
└─ Prevents "I was hacked" excuses

USAGE:
  // Submit code
  CodeAudit::log_submission($attempt_id, $userid, $code);
  
  // Later, verify code hasn't changed
  $integrity = CodeAudit::verify_integrity($attempt_id, $code);
  if (!$integrity['valid']) {
    echo "⚠️ CODE WAS MODIFIED!";
  }

EXAMPLE:
  Student submits:  print("1+1=2")
  Hash created:     a1b2c3d4e5f6...
  Later, student changes to: print("1+1=3")
  Hash no longer matches!  ❌ CHEATING DETECTED

═══════════════════════════════════════════════════════════════════
PROBLEM 2: EXTERNAL ATTACKER SUBMITS MALICIOUS CODE
═══════════════════════════════════════════════════════════════════

SOLUTION: CodeValidator.php
├─ Blocks dangerous imports (os, subprocess, socket, etc.)
├─ Blocks dangerous functions (exec, eval, compile, etc.)
├─ Prevents shell command execution
└─ Detects code obfuscation

USAGE:
  $validation = CodeValidator::validate($student_code);
  
  if (!$validation['is_valid']) {
    echo $validation['errors'][0];  // "Cannot import 'os'"
  }

EXAMPLES OF BLOCKED CODE:
  ❌ import os           → Cannot import 'os' module
  ❌ exec('whoami')      → Cannot use 'exec()' function
  ❌ subprocess.run()    → Cannot import 'subprocess'
  ❌ socket.connect()    → Cannot import 'socket'
  ✅ print(1+1)         → OK - Safe code

═══════════════════════════════════════════════════════════════════
PROBLEM 3: ATTACKER SPAM / DENIAL OF SERVICE
═══════════════════════════════════════════════════════════════════

SOLUTION: Rate Limiting (in SecureCodeSubmissionHandler)
├─ Max 10 submissions per minute per user
├─ Max 50 requests per minute per IP
├─ 5 second execution timeout
└─ Max 50KB output size

ATTACK EXAMPLES PREVENTED:
  ❌ Attacker sends 1000 requests/second → Blocked after 10/min
  ❌ Attacker sends: while True: pass → Timeout after 5 seconds
  ❌ Attacker tries to crash server → Memory limited, timeout stops it
