## 2️⃣ WHERE MARKS ARE STORED
══════════════════════════════════════════════════════════════════════════════

QUIZ GRADES STORAGE:
────────────────────────────────────────────────────────────────────────────────

DATABASE TABLES (MariaDB):

1. mdl_quiz_attempts
   ├─ id                    (Attempt ID)
   ├─ quiz                  (Quiz ID)
   ├─ userid                (Student ID)
   ├─ attempt               (Attempt number: 1st, 2nd, etc.)
   ├─ sumgrades             (Raw marks/points)
   ├─ timeanswer            (Timestamp of answer)
   └─ state                 (finished, inprogress, abandoned)

2. mdl_question_attempts
   ├─ id
   ├─ questionusageid       (Links to usage)
   ├─ questionid            (Question ID - like Question 20)
   ├─ maxmark               (Total marks for question)
   ├─ slot                  (Question slot in quiz)
   └─ responsesummary       (Student's code submission)

3. mdl_question_attempt_steps
   ├─ id
   ├─ attemptid             (Links to question_attempts)
   ├─ sequencenumber        (Step in question: 0=initial, 1=first answer, etc.)
   ├─ state                 (complete, todo, invalid)
   ├─ timecreated           (When step was created)
   └─ questionid            (Question ID)

4. mdl_grade_grades (Overall course grades)
   ├─ id
   ├─ userid                (Student)
   ├─ itemid                (Quiz)
   ├─ rawgrade              (Mark before scaling)
   ├─ finalgrade            (Final percentage)
   ├─ timemodified          (Last updated)
   └─ feedback              (Teacher feedback)

═════════════════════════════════════════════════════════════════════════════

SQL QUERY TO VIEW MARKS:
────────────────────────────────────────────────────────────────────────────────

SELECT SQL:
──────────

-- View all student marks for a quiz
SELECT 
    u.id as student_id,
    u.firstname,
    u.lastname,
    qa.attempt as attempt_number,
    qa.sumgrades as marks_obtained,
    q.grade as total_marks,
    (qa.sumgrades / q.grade * 100) as percentage,
    FROM_UNIXTIME(qa.timeanswer) as submission_time
FROM mdl_quiz_attempts qa
JOIN mdl_quiz q ON qa.quiz = q.id
JOIN mdl_user u ON qa.userid = u.id
WHERE q.id = 2  -- Replace with your quiz ID
ORDER BY u.id, qa.attempt;

Question-wise breakdown:
────────────────────

SELECT 
    u.firstname,
    u.lastname,
    q.name as question_name,
    qas.maxmark as total_marks,
    qas.summarks as marks_obtained,
    (qas.summarks / qas.maxmark * 100) as question_percentage
FROM mdl_question_attempts qas
JOIN mdl_question q ON qas.questionid = q.id
JOIN mdl_user u ON qas.userid = u.id
WHERE qas.id IN (
    SELECT attemptid FROM mdl_question_attempt_steps 
    WHERE state = 'complete'
)
ORDER BY u.id, q.name;

## 4️⃣ PROTECT STUDENT DATA: Recommended Settings
══════════════════════════════════════════════════════════════════════════════

🔒 TO MAKE IT MORE SECURE:
────────────────────────────────────────────────────────────────────────────────

A. DATABASE PROTECTION
──────────────────────

Edit: config.php
────────────────

Add these lines:

// Protect student data from public access
$CFG->passwordpolicy = 1;               // Enforce strong passwords
$CFG->passwordpolicylowercasenames = 1; // Require lowercase
$CFG->passwordpolicynumbersrequired = 1;// Require numbers
$CFG->passwordpolicypolicylength = 8;   // Min 8 characters

// Secure session handling
$CFG->sessiontimeout = 3600;             // 1 hour timeout
$CFG->cookiehttponly = 1;                // HTTP only cookies
$CFG->cookiesecure = 1;                  // HTTPS only (production)

// Prevent data leakage
$CFG->enablecsvimport = 0;               // Disable CSV export
$CFG->enablerssfeeds = 0;                // Disable RSS feeds
$CFG->enableapi = 0;                     // Disable API access (if not needed)

// Hide user information
$CFG->hiddenuserfields = array(
    'city',
    'country',
    'email',
    'phone1',
    'phone2'
);

// Grade visibility
$CFG->gradebookroles = array(3, 4);      // Only teachers (3) and admins (4)

## 8️⃣ WHERE MARKS GO (Visual Flow)
══════════════════════════════════════════════════════════════════════════════

Student Submission Flow:
─────────────────────────────────────────────────────────────────────────────

    [Student writes code] 
           ↓
    [CodeValidator.php]  ← Checks for dangerous code
           ↓ (if valid)
    [Code executed]
           ↓
    [Output compared] ← Check against expected output
           ↓
    [Marks calculated] ← e.g., 1/1 = 100%
           ↓
    [CodeAudit.php] ← SHA256 hash + log created
           ↓
    [Database Storage]:
       - mdl_quiz_attempts  ← Grade for quiz
       - mdl_question_attempt_steps ← Grade for question 20
       - mdl_code_audit ← Full audit trail
           ↓
    [Moodle Gradebook]
       - mdl_grade_grades ← Final course grade
           ↓
    [Student Views] ← Via Moodle UI (if enabled in review options)
                    ← Cannot access database directly
                    ← Cannot see other students' marks
                    ← Cannot modify marks
