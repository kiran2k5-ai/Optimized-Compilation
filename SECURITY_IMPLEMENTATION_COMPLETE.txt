LAYER 2: SUBMISSION AUDITING & TAMPERING DETECTION (CodeAudit.php)
===================================================================
✓ Creates SHA256 hash of every submission (cryptographic proof)
✓ Stores submission timestamp, IP address, user agent
✓ Detects code tampering after submission
✓ Tracks complete version history
✓ Detects suspicious patterns (rapid changes, large size jumps)
✓ Prevents student cheating (code can't be modified after grading)

Usage:
  // When code submitted:
  CodeAudit::log_submission($attempt_id, $userid, $code);
  
  // Later, verify code wasn't modified:
  $integrity = CodeAudit::verify_integrity($attempt_id, $code);
  if (!$integrity['valid']) {
    echo "⚠️ Code has been tampered with!";
  }
  
  // Check for suspicious behavior:
  $suspicious = CodeAudit::detect_suspicious_activity($attempt_id);
  if ($suspicious['suspicious']) {
    foreach ($suspicious['issues'] as $issue) {
      log_security_alert($issue);
    }
  }

Example detection:
  Student submits: "print(1)" [Hash: abc123]
  Later database shows: "print(999)" [Hash: xyz789]
  Verification returns: ❌ "Code has been modified since submission"

SUPPORT & QUESTIONS
===================

If code is blocked but it's legitimate:
→ Check CodeValidator.php SAFE_MODULES
→ Add safe imports to whitelist (e.g., 're', 'string', 'math')
→ Contact security team before allowing dangerous imports

If you need to allow more imports:
→ Edit CodeValidator.php
→ Add to SAFE_MODULES: const SAFE_MODULES = ['...', 'new_module']
→ Restart web server
→ Re-test

If production performance is slow:
→ Reduce SAFE_MODULES list (less regex matching)
→ Archive old audit logs (> 6 months)
→ Add database indexes on (attempt_id, timestamp)
→ Consider moving to dedicated logging server
