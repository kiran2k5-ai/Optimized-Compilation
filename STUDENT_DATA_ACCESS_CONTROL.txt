STUDENT DATA ACCESS CONTROL - DETAILED GUIDE
==============================================

## HOW MOODLE PROTECTS STUDENT DATA
═════════════════════════════════════════════════════════════════════════════

QUESTION: "Can students access their marks while moving to next question?"
ANSWER: ✅ YES (by default) - Marks appear immediately after submitting each question

QUESTION: "Can students be access to students?" (Access other students' marks)

## PART 3: HOW MOODLE ENFORCES THIS PROTECTION
═════════════════════════════════════════════════════════════════════════════

MECHANISM 1: Database Query Filtering
─────────────────────────────────────────────────────────────────────────────

When STUDENT views grades:
├─ Moodle retrieves current user ID: $USER->id = 5
├─ Query becomes: SELECT * FROM mdl_grade_grades WHERE userid = 5
└─ Result: Only THAT STUDENT's marks shown

WHY THEY CAN'T SEE OTHERS:
├─ Moodle code ALWAYS adds: WHERE userid = $USER->id
├─ Database never returns rows where userid ≠ 5
└─ Student never receives data for userid 6, 7, 8, etc.

Example code (in Moodle):
───────────────────────
$sql = "SELECT * FROM {grade_grades} WHERE userid = ?";
$grades = $DB->get_record_sql($sql, array($USER->id));
                                           ↑ Student's ID only

MECHANISM 2: Permission Checks
─────────────────────────────────────────────────────────────────────────────

Before showing any grade data:
├─ Check: does_this_user_have_viewallgrades_permission()?
├─ Student (Role 5): viewallgrades = 0 (NO)
├─ Teacher (Role 3): viewallgrades = 1 (YES)
├─ Admin (Role 1): viewallgrades = 1 (YES)
└─ If NO permission → Moodle redirects with "Access Denied"

MECHANISM 3: Context-Based Isolation
─────────────────────────────────────────────────────────────────────────────

Student can only access:
├─ Quiz context (where they are enrolled)
├─ Their own course (where they are a student)
└─ Their own submissions/grades

Cannot access:
├─ Other courses they're not in
├─ Admin contexts (system, admin area)
└─ Teacher-only data

## PART 4: HANDS-ON TEST - VERIFY SECURITY
═════════════════════════════════════════════════════════════════════════════

TEST 1: Can Student See Other Student's Marks?
─────────────────────────────────────────────────────────────────────────────

Test Steps:
───────────
1. Login as Student 1 (username: student1)
2. Go to: Course → Grades
3. Look for: Can you see Student 2's marks?
   Expected Result: ✗ NO (only own marks visible)

What happens internally:
┌─────────────────────────────────────────────────┐
│ Student says: "Show me grades for course 2"     │
│ Moodle checks: Can this user see grades?        │
│ Moodle finds: role = STUDENT                    │
│ Moodle checks: STUDENT permission for viewgrades│
│ Result: permission = "own grades only"          │
│ Moodle filters: Add WHERE userid = {current_user}
│ Database query: SELECT * FROM mdl_grade_grades  │
│                 WHERE userid = 5 AND itemid = 4 │
│ Result: Only records for userid=5 returned     │
└─────────────────────────────────────────────────┘

TEST 2: Can Student Access Database Direct?
─────────────────────────────────────────────────────────────────────────────

Test Steps:
───────────
1. Try to access PhpMyAdmin: http://localhost/phpmyadmin
2. Try to login with student credentials
   Expected Result: ✗ FAIL (no credentials given to students)

Why it fails:
├─ Students don't have MySQL username/password
├─ config.php stores credentials (hidden from students)
├─ Web server only accesses database via PHP
├─ PHP validates user role before any query
└─ Database never receives requests from student directly

TEST 3: Can Student Modify Their Grade in Database?
─────────────────────────────────────────────────────────────────────────────

Test Steps:
───────────
1. Even if they could access database
2. Student has NO UPDATE permission on mdl_quiz_attempts
3. Only Teacher/Admin can execute UPDATE queries

Protection layers:
├─ Layer 1: No database access (no credentials)
├─ Layer 2: Even with access, no UPDATE permission
├─ Layer 3: Code Audit tracks modification (SHA256 hash)
├─ Layer 4: Teacher email alert on grade changes
└─ Layer 5: Audit log shows who changed what when

## PART 5: VIEW STUDENT MARKS AS ADMIN
═════════════════════════════════════════════════════════════════════════════

HOW TO VIEW ALL STUDENT MARKS (Admin Only):
─────────────────────────────────────────────────────────────────────────────

Method 1: Via Moodle UI
───────────────────────
1. Login as Admin
2. Go to: Course → Grades
3. You will see grid:
   
   Student Name │ Q20 │ Q21 │ Q22 │ ... │ Total
   ─────────────┼─────┼─────┼─────┼─────┼──────
   John Smith   │ 1   │ 1   │ 0   │ ... │ 5/10
   Jane Doe     │ 1   │ 0   │ 1   │ ... │ 4/10
   Mike Brown   │ 0   │ 0   │ 0   │ ... │ 0/10

Method 2: Via Database Query (SQL)
──────────────────────────────────
Run this query:

SELECT 
    CONCAT(u.firstname, ' ', u.lastname) as 'Student',
    q.name as 'Quiz',
    qa.sumgrades as 'Marks',
    q.grade as 'Total',
    ROUND((qa.sumgrades/q.grade)*100, 2) as 'Percentage',
    FROM_UNIXTIME(qa.timeanswer) as 'Submitted'
FROM mdl_quiz_attempts qa
JOIN mdl_quiz q ON qa.quiz = q.id
JOIN mdl_user u ON qa.userid = u.id
WHERE q.id = 2  -- Quiz ID
ORDER BY u.firstname;

Method 3: Generate Report
───────────────────────────
1. Go to: Admin → Reports → Quiz Statistics
2. Select Quiz: "Addition"
3. View detailed breakdown

## PART 7: DATA SECURITY SUMMARY
═════════════════════════════════════════════════════════════════════════════

STUDENT DATA PROTECTION CHECKLIST:

[✓] Database Access
    ├─ Students: NO direct database access
    ├─ Access via: Web server → PHP → Database
    ├─ Credentials: Hidden in config.php
    └─ Role: Read-only to Moodle app, no raw SQL

[✓] Grade Visibility
    ├─ Student sees: Only OWN grades
    ├─ Database enforces: WHERE userid = student_id
    ├─ UI enforces: Permission checks before display
    └─ Logs: All access logged in mdl_log table

[✓] Grade Modification Protection
    ├─ Students: CANNOT update grades
    ├─ Teacher: CAN update (with audit trail)
    ├─ Modification: Logged with timestamp + teacher ID
    └─ Detection: CodeAudit logs all changes

[✓] Data Transmission
    ├─ Local (XAMPP): HTTP (acceptable for testing)
    ├─ Production (Render): HTTPS (encrypted)
    ├─ Data in transit: Cannot be intercepted
    └─ Session: Secure cookies, timeout = 1 hour

[✓] Audit Trail
    ├─ All submissions: Logged with SHA256 hash
    ├─ All grades: Tracked with modification timestamps
    ├─ All access: Recorded in mdl_log
    └─ Reports: Available to admin only
